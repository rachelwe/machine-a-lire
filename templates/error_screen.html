Cela peut venir de plusieurs points, assurez-vous donc de vérifier les éléments suivants l'un après l'autre (par ordre de probabilité) :

{% include "errors_list.html" %}
  1. C'est volontaire : La machine est allumée pour la 1re fois ou vous venez de supprimer avec succès tous les fichiers avec la clé SUPPR (bravo !).
  2. La page n'est pas à jour : Cliquez sur le bouton "Recharger la page" en haut OU redémarrez la machine (en enlevant bien les clés USB avant).
  3. Un ou plusieurs des fichiers ne respectent pas la nomenclature et le formatage des fichiers textes : rendez-vous sur doc-machine-a-lire.webtopie.fr/v2 pour vous aider lors de la création et retrouver les éléments et indications de format obligatoires.
  4. La clé usb d'ajout ou de suppression est restée branchée lors du redémarrage de l'appareil : Cela peut créer des conflits. Pour résoudre :
    1. Option 1 : débrancher les éventuelles clés USB → redémarrer. Après le redémarrage terminé : enclenchez la clé "AJOUT". Lorsque la notification "Traitement terminé !" apparaît (cela peut prendre plusieurs minutes), enlevez la clé AJOUT et cliquez sur le bouton "Recharger la page" en haut OU redémarrez la machine.
    2. ⚠️ Intervention technique, ne pratiquer qu'avec assistance de l'ALCA ⚠️
      Option 2 si l'option 1 n'a pas fonctionné : débrancher les éventuelles clés USB, puis brancher un clavier, appuyer sur les touches alt+f4, puis lancer un terminal. Dans le terminal (carré noir en haut à gauche avec les symboles ">_"), exécuter les commandes suivante : "cd /media/pi", entrer, puis "sudo rm -r AJOUT/", entrer. Redémarrer le raspberry puis retenter le processus d'ajout (insérer la clé USB, attendre la notification de traitement terminé et cliquer sur recharger la page).
